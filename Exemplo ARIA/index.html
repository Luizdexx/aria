Exemplo de página acessível com ARIA

Seção 1

Um acordeão é um conjunto de seções de conteúdo interativas empilhadas verticalmente. Cada seção possui um título,
que funciona como um controle para revelar ou ocultar o conteúdo.
Acordeões são usados ​​para reduzir a necessidade de rolagem em páginas com várias seções.
Os termos para entender acordeões incluem cabeçalho (título) e painel (conteúdo).
Alguns acordeões podem ter elementos adicionais, como botões de menu ou trechos de conteúdo visíveis.

Seção 2

WAI-ARIA suporta dois tipos de widgets de caixa de seleção :
caixas de seleção de estado duplo alternam entre duas opções - marcadas e não marcadas - e caixas de seleção de três estados,
que permitem um terceiro estado adicional conhecido como parcialmente verificado.
Um uso comum de uma caixa de seleção de três estados pode ser encontrado em instaladores de software onde uma única caixa de seleção de três estados
é usada para representar e controlar o estado de um grupo inteiro de opções de instalação.
E cada opção do grupo pode ser ativada ou desativada individualmente com uma caixa de seleção de estado duplo.

Seção 3

Um alerta é um elemento que exibe uma mensagem breve e importante de forma a atrair a atenção do usuário sem interromper sua tarefa.
Alertas renderizados dinamicamente são anunciados automaticamente pela maioria dos leitores de tela e, em alguns sistemas operacionais, podem acionar um som de alerta.
Um botão é um widget que permite aos usuários acionar uma ação ou evento, como enviar um formulário, abrir uma caixa de diálogo, cancelar uma ação ou executar uma operação de exclusão.
Uma convenção comum para informar aos usuários que um botão inicia uma caixa de diálogo é anexar "…" (reticências) ao rótulo do botão, por exemplo, "Salvar como…".
